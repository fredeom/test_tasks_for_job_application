1) Необходимо создать 2 приложения. В приложении `creator` создается сообщение и хранится в очереди на отправку. Сообщение имеет статус приоритетности `info`, `warning`, `error`.
Приложение `gateway` отправляет сообщение адресату (проксирует) и отвечает 2 статусами `success`, `error`. Приложение `creator`, в зависимости от ответа шлюза, принимает решение о переотправке сообщения. Сообщения переотправляются не чаще чем раз в минуту. 
Предусмотреть:
1.1) Максимальное число попыток переотправки:
Сообщения типа `info` - 3 раза.
Сообщения типа `warning` - 5 раз.
Сообщения типа `error` - 8 раз.
1.2) Отправку по приоритету
1.3) Отправку разными способами email,смс (непосредственную отправку можно не реализовывать).

2) Задание по sql
2.1) Необходимо создать таблицу `numbers`, содержащие поля
`id`,`num`, `parentId`
Поля `id` и `num` являются уникальными.
2.2) Заполнить таблицу 10-20 записями (заполнять колонки `id`,`num`)
2.3) Написать sql запрос который бы проставил parentId. ParentId определяется по возрастанию (то есть, у максимального элемента `num` parentId = NULL). 

UPDATE test t0 INNER JOIN test t1 ON t1.num > t0.num 
LEFT JOIN test t2 ON t2.num < t1.num AND t0.num < t2.num
set t0.parentId = t1.id
where t2.id is null
